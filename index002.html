CSS 적용 안하면 테두리가 아무것도 안보인다

td 사이에 tr 쓰면 위에 써지고 줄바꿈도 된다
table tr th td
사용 제한 table이 작성되는 시작 부분 tr로 줄바꿈 한다라고 생각하면 편할 것 같다 태그들은 대부분이 사용 제한이 없다 이것도 그렇다
사용 목적 " 표라는 형식에 하위 글이 포함되게 한다 th는 표의 제목, 헤더 td는 표에 넣을 내용을 넣을 때 쓴다
임의로 tr 썻더니 줄바꿈 된 위치에서 시작하는 td
사용 위치 1. 표 하나당 table 하나다
2. 테이블 여러개 만들면 가장 큰 영역의 사각형도 여러개 생긴다
3. 지금 이것도 테이블로 묶여서 tr 이상하게 써도 하나로 묶여있는 것만 		1. 줄을 구분하고 영역별로 인덱스 구분하기 위해 쓴다 2. 없이 써도 tr 닫힌 다음 td 쓰면 한 줄까지는 줄바꿈이 된다 그 이후는 안되니까 tr을 쓰겠지 th과 tr 모두 colspan, rowspan 속성을 쓸 수 있다
속성 X X scope
colspan, rowspan

테이블 병합 특징 <... colspan="">, <... rowspan"">

테이블에서 colspan과 rowspan 은 직선적이다

각각의 칸들이 좌표를 가진다고 생각했을 때
(2,0) 에서 colspan="3"을 쓰면 (5,0)까지 사용되고 (2,0) 에서 rowspan="3"을 쓰면 (2,2)까지 사용된다 (1,1)(1,2)은 비워진 상태로 (2,2)만 생긴다 단 최소 하나의 표가 있어야 활성화 된다
